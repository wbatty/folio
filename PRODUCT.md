# Product

## Register

product

## Users

Single user: a job seeker running an active search. Three distinct sessions
daily — morning intake (adding jobs, scanning emails), daytime deep work
(referencing scraped job data, working with the AI on applications), and
evening review (pulling reports for reflection). Laptop-primary, natural
light in the morning, probably dimmer by evening.

## Product Purpose

Folio is a personal job application cockpit. It centralises every stage of
a search — from scraping a posting to AI-assisted cover letter work to
pipeline tracking — so the user never has to hold the job search in their
head. Success means ending each day knowing exactly where every application
stands, with nothing dropped.

## Brand Personality

Methodical, efficient, direct. The emotional goal is calm reassurance: a
tool that absorbs the anxiety of the job search rather than amplifying it.
Minimal, but warm — not stark. Think a well-organised workspace, not a
sterile dashboard.

## Anti-references

LinkedIn, Indeed, Facebook — avoid the corporate-blue social-platform
aesthetic: notification-badge anxiety, metric-heavy feeds, aggressive CTAs,
noisy sidebars. Folio is not trying to sell the user anything.

## Design Principles

1. **Cockpit, not feed** — every element has a job; nothing decorative. The
   view is organised around the current task, not a stream of activity.
2. **Calm authority** — information is presented with confidence and
   completeness, never urgency. Status never shouts.
3. **Daily rhythm** — the interface respects the morning/day/evening cadence:
   intake, deep work, reflection. Hierarchy shifts to match the mode.
4. **Warmth without softness** — warm neutrals and earthy tones over cold
   grays, but the layout stays tight and purposeful.
5. **Direct, never cold** — labels, copy, and states are plain-spoken. No
   jargon, no hedging, no decorative prose.

## Accessibility & Inclusion

No specific requirements at this time. Standard semantic HTML and keyboard
navigation as baseline.
